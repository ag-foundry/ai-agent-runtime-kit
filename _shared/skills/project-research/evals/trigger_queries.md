# Trigger evaluation set — project-research

## Should trigger
1. Create a plan for a new website that compares competitors and chooses a stack.
2. Design a sales analytics workflow with external dashboards and source systems.
3. Plan a new automation service that syncs data between two platforms.
4. Explore approaches for a document-processing pipeline with uncertain tooling.
5. Start a new infrastructure project that depends on a third-party platform.
6. Evaluate options for a new business workflow system with integration risks.

## Should not trigger
1. Fix a typo in README.md.
2. Rename one variable in an existing script.
3. Continue implementing step 3 of an already approved project plan.
4. Update one local config value with no external dependency change.
5. Summarize the current LOG.md.
6. Reformat an existing JSON file.
